Nginx安装

在官网找到你需要的版本并且用wget下载，这里选择一个合适的目录，我选择的是/usr/local,并且用的是nginx稳定版1.14.0，之后编译安装，访问服务器，就能看到nginx的欢迎界面了。具体操作如下：

cd /usr/local   #cd到路径下
wget https://nginx.org/download/nginx-1.14.0.tar.gz #下载nginx
tar -zxvf nginx-1.14.0.tar.gz  #解压
cd nginx-1.14.0  #cd到文件夹里
./configure --prefix=/usr/local/nginx  #配置编译路径
make & make install  #编译安装
#这样/usr/local/nginx就是nginx的整个文件目录了
/usr/local/nginx/sbin/nginx #启动nginx，在从外部访问你的服务器，你就能看到nginx的欢迎界面了

当然这样有个局限，每次要启动或者关闭我们都要写一个绝对路径(/usr/local/nginx/sbin/nginx )才能找到nginx，为了方便，我们把nginx加入到全局环境变量：

xxxxxxxxxx
vi /etc/profile     #打开文件，加入下面两行代码
export NGINX_HOME=/usr/local/nginx
export PATH=$PATH:$NGINX_HOME/sbin
#保存之后执行下面一行代码
source /etc/profile

这样下次就直接输入 nginx 就可以执行nginx命令了

 

Nginx 配置 uWsgi

我们在项目中应该会有一个uWsgi文件来代理我们的web服务，并且我们选择了一个端口用来和Nginx通讯。

我们启动Nginx时，Nginx就接管了配置过的端口（默认80），这就是为什么启动Nginx后访问80端口进入的是一个Nginx欢迎界面。

我们在nginx的配置文件——nginx.conf一般在nginx目录下的conf目录中，比如刚才我们讲Nginx安装到了/usr/local/nginx下，你就能在/usr/local/nginx/conf/下看到这个conf文件。

主要看server块，可以有很多个server，每一个server监听一个端口，里面的结构主要是

xxxxxxxxxx
server{
    。。。 #全局配置
    location /{
        。。。#默认路径
    }
    。。。#其他
}

以刚才我们的uwsgi配置文件为例子，我们用了8002端口Nginx通讯，并且我们的项目文件在/home/web/project2下面，所以server应该是这样的

xxxxxxxxxx
server{     
        listen       80 default_server;
        listen       [::]:80 default_server;
        server_name  _;
        root         /home/web/project2;
​
        location / {
            include uwsgi_params;
            uwsgi_pass 127.0.0.1:8002;
        }
}

改成这样，重启nginx，开启uwsgi项目，就能够通过nginx访问到我们的项目了。

只不过这样的配置是最简单的配置，如果想要自定义配置，可以看上面完整的nginx配置文件，根据自已需求，增加相应的配置就可以了。
